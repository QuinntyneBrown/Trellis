Menus

One raised-list idiom covers everything that floats from a trigger: the rail's hamburger application menu and the template picker's pop-out catalog. A raised surface, strong hairline, soft shadow, and quiet hover rows.

Derived from: templates/template-picker/ pop-out panel (the app's raised-list reference); adopted by the rail's application menu — .tds-menu.

Application menu (rail hamburger)

Trigger from the rail's hamburger button (editor-toolbar); a flyout of the app menus, vscode.dev-style — File's submenu carries its command set.

The hamburger stays filled while its menu is open; File's entry keeps its hover fill while its submenu is expanded (second submenu item shown hovered). The submenu opens on hover or click; hovering another entry collapses it. Edit/View/Help are chevroned but inert today.

Template picker pop-out

Derived from: template-picker__panel/__option — the catalog flying out from the rail's Templates button.

Names and categories match the backend's static template catalog; category renders as the uppercase hint.

Command center

Derived from: title-bar__command-center. Documented with the title bar in Toolbars & chrome; noted here because it is menu-adjacent: a pill-shaped trigger, not a menu itself.

Rules

Do Right-align supplementary text (shortcuts, categories) as the uppercase muted hint — one hint per item at most.
Do Return focus to the trigger when a menu dismisses.
Don't Nest beyond one submenu level — only the application menu's top-level entries (File ▸ …) open submenus; everything else stays one level deep.
Don't Put icons in menu items; the raised list is a text idiom (the submenu chevron is the one navigational affordance allowed).
