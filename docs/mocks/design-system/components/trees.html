Trees

One tree-row language for both trees: the Explorer's disk files and the Documents panel's virtual folders share a single style source, so a row reads identically everywhere — chevron (or spacer), truncating name, always-visible icon actions. Folders sort first, then leaves, case-insensitive.

Drag and drop

Document rows are draggable. A hovered folder row takes the drop-target state; the tree's empty space below the last row is the move-to-root zone and takes the panel-level wash (.tds-panel__content--drop-target, dashed --tds-accent-soft). The destination folder auto-expands after a move. The row's "Move to Folder…" dialog is the keyboard fallback — drag is never the only path.

Rules

Do Sort folders first, then leaves, case-insensitive — both trees, no exceptions.
Do Keep row actions always visible; hover-revealed actions hide functionality from keyboard and touch users.
Don't Add second lines to rows — metadata lives in the title tooltip, keeping the tree dense.
Don't Use the active-document treatment for selection; it means "open in the editor", nothing else.
