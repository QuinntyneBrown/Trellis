Form controls

Trellis has a deliberately tiny form vocabulary: a text input, a select, and a label — all living inside dialogs — plus a hidden file input behind the rail's Upload button. Free-form prompts (rename, new folder) use the browser's native window.prompt, not custom UI.

File upload

Derived from: editor-toolbar__upload-input (hidden <input type="file">).

There is no drop-zone or upload widget: the rail's Upload button forwards its click to a visually hidden file input (display: none), and the browser's native picker takes over. The button is the entire visible affordance.

<input type="file" accept=".puml,.plantuml,.txt" hidden>

Native dialogs

Create-folder and rename flows use window.prompt; deletes use window.confirm (a folder delete warns that contents are deleted too). These are contractual: no custom prompt UI exists, so those flows inherit the browser's own styling and accessibility.

Rules

Do Pair every control with a visible .tds-field-label bound via for/id — placeholder text is never the label.
Do Mark invalid fields with aria-invalid="true" alongside the error border.
Don't Grow the form vocabulary (checkboxes, radios, textareas) without a design-system addition here first — the small surface is deliberate.
Don't Rebuild the native select popup or file picker; only the closed control is themed.
