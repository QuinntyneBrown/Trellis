Buttons

Three button families, each owned by a context: 40×40 icon buttons in the activity rail, compact 1.4rem icon actions inside tree rows and panel headers, and text buttons in dialogs and panels. All share the same focus contract: a 2px accent ring at −2px offset.

Rail button

Derived from: shared/components/rail-button/ — .tds-rail-button.

The activity rail's icon button: 40×40 hit target, 20px glyph, hover/focus-revealed tooltip flying out to the right, and — when its panel or mode is active — the accent color plus a 3px accent edge bar hugging the rail's left edge.

Choosing a button

Do Use the danger variant for every destructive row action — delete is red before it is clicked, not after.
Do Keep one primary action per dialog; a dialog with two blue buttons has no primary action.
Don't Put text buttons in the rail or icon-only buttons in dialog action rows — the families don't cross contexts.
Don't Ship an icon button without aria-label (and title for row actions).
