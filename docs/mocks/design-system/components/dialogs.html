Dialogs

One modal pattern, four occupants. Save, New Document, Move to Folder, and Export Folder all render the same anatomy — full-viewport scrim, centered overlay panel, title, fields, right-aligned actions — so the modals read as one component with different contents.

Rules

Do Keep dialogs one-question small. A dialog asks for a name and/or a destination — anything bigger belongs in a panel.
Do Name the thing being acted on in the title ("Move "sequence-checkout"") so the scrim never hides the context.
Don't Stack dialogs, add close-X buttons (Cancel is the exit), or animate entry/exit.
Don't Rebuild confirm/prompt flows as custom dialogs — deletes and renames contractually use window.confirm / window.prompt.
