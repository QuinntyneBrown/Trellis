Diagram preview

The preview pane walks a four-state lifecycle: placeholder → rendering → rendered → error. Renders are requested over the SignalR hub and keyed by a render sequence number, so a stale response can never overwrite a newer one.

Rendering

.tds-preview__spinner-overlay — the previous render stays visible; the spinner pins to the top-right corner.

A re-render in flight over the previous result — content is never blanked while waiting.

Rendered

.tds-preview__svg — the PlantUML SVG, centered on a white plate.

The artwork keeps PlantUML's own light styling (#E2E2F0 shapes, #181818 lines) on a white plate with a soft shadow — exports must stay legible outside the app, so diagrams are never re-themed dark.

Error

.tds-preview__error — the friendly message built from PlantUML's stderr (PlantUmlRenderer.BuildFriendlyErrorMessage), pre-wrap preserved.

Rendering failed (line 4): Syntax error: participant "Orders API as API Expected a closing quote.

ErrorHub replies isSuccess: falseError banner treatment with the friendly stderr message; previous SVG is replaced

Responses carrying a stale data-render-seq are discarded — the newest request wins, regardless of arrival order.

Rules

Do Keep the last good render visible during re-renders; flashing to blank makes iteration feel slower than it is.
Don't Re-color the rendered SVG to match the dark theme — the white plate is the contract with exported artwork.
