Workbench shell

The composition every Trellis screen lives in: title bar on top, then a body of activity rail → optional side panel → editor pane → resize seam → preview pane. The shell is fixed chrome; only the panel width and the editor/preview ratio move.

Shell behaviors

  • Application menu. The hamburger atop the rail opens the app menus (File / Edit / View / Help) as a flyout to its right, vscode.dev-style — see Menus.
  • Panel toggling. A rail button opens its panel or closes it if already active; the title bar's primary-sidebar toggle mirrors the same state with its filled band.
  • Resizing. Dragging a seam resizes live with no animation; during a drag the body carries cursor: col-resize and suppresses text selection (body.is-resizing-panes).
  • Keyboard. Ctrl+Enter renders; Ctrl+S saves (opening the save dialog for untitled documents).
  • State reporting. At rest the command center names the open document ("Untitled diagram — Trellis" before first save); activating it (click, or Ctrl+P) swaps the pill for the Quick Open search. The connection dot is the single connection indicator.

Rules

Do Keep every screen inside this frame — new features land as panels, dialogs, or preview content, not as new top-level layouts.
Don't Add horizontal toolbars between the title bar and the panes; the rail and the title bar are the only chrome.
