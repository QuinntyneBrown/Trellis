Editor surface

The editing pane hosts Monaco. The app configures it minimally — plaintext language, minimap off, no scroll beyond the last line — and the design system pins the theme to vs-dark so the surface sits flush with the workbench. These classes document the surface contract; Monaco draws the real thing at runtime.

Editing behaviors

  • Ctrl+Enter — render the current source to the preview pane.
  • Ctrl+S — save; untitled documents get the save dialog, disk files write through their file handle (failures raise the disk-save toast).
  • Loading a template or document replaces the buffer; the app confirms via native dialogs when unsaved changes would be lost.

Rules

Do Keep the editor's background exactly on the content surface — a seam, not a color change, separates it from the preview.
Don't Re-style Monaco internals beyond the theme; the surface contract (metrics, gutter, caret) is the integration point.
