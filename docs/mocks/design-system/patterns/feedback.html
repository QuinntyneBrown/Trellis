Feedback & empty states

How Trellis communicates "nothing here yet", "something went wrong", and "I need your permission again". Empty states teach the next action; errors state what happened and what to do; permission flows re-use the panel button so recovery looks like any other action.

Explorer — no folder yet

Derived from: features/explorer/explorer-panel/ — .tds-button--panel.

Explorer
You have not yet opened a folder.

The full-width panel button is the VS Code-style call to action. Opening a folder uses the File System Access API (showDirectoryPicker), so it is only offered in browsers that support it.

Permission re-grant

Derived from: features/explorer/ permission handling — same empty-state + panel-button grammar.

Explorer
Permission to diagrams/ has expired.

Folder handles persist in IndexedDB across sessions, but the browser may drop read permission. The panel names the remembered folder and offers a single re-grant action that calls requestPermission() — never an error banner, because nothing is broken.

Disk-save error toast

Derived from: features/editor/ disk-save error toast — .tds-toast-anchor + .tds-error-banner.

Failed to save "checkout-flow.puml" to disk: the file may be locked by another program.

Fixed top-center (--tds-z-toast), role="alert" so it is announced on appearance. The editor buffer keeps the user's changes — the toast reports the disk write only.

Connection loss

Derived from: core/connection-status — .tds-status-dot in the title bar.

disconnected reconnecting connected

SignalR reconnects automatically; the dot narrates it. Renders requested while disconnected fail into the preview's error state — the shell never blocks input on connectivity.

Native confirms

Destructive and buffer-replacing actions confirm through native window.confirm / window.prompt — delete document, delete folder (names its contents), rename, and "discard unsaved changes?" when loading over a dirty buffer. The design system deliberately does not restyle these: a browser-owned dialog cannot be spoofed by page content.

Choosing a channel

SituationChannel
Nothing to show yetPanel empty state (+ panel button when there's an action)
Background operation failedToast (error banner, fixed top-center)
Render failedPreview error state, in place
About to lose dataNative confirm
ConnectivityStatus dot; no modal, no toast

Rules

Do Pair every empty state with its exit: a keyboard hint or a panel button.
Don't Use an error treatment for expired permissions or empty trees — reserve red for operations that actually failed.
