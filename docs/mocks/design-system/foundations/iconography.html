Iconography

Hand-authored outline glyphs on a shared 24×24 viewBox — fill: none; stroke: currentColor, 1.75 stroke width, round caps and joins (a Heroicons/Feather-like style). There is no icon-library dependency anywhere in Trellis.

Conventions

  • One color. Icons inherit currentColor from their button; state is expressed by the button's color, never by multi-tone artwork.
  • Two closed sets. The rail set and the row-action set are independently owned by their consuming components (rail-icons.ts, tree-action-icons.ts) and deliberately not merged — the 40px rail button and the dense 1.4rem row action have different size and tooltip mechanics. Shared drawings (page-with-plus) are copied, not imported.
  • Rendered sizes: 20px in the rail, 14px in row actions, 18px for the title-bar logo.
  • Visually distinct neighbors. Glyphs that sit a few pixels apart in the rail (new / documents / explorer) use very different silhouettes so they never blur together.

The logo is a trellis lattice (crossed horizontals and verticals) drawn in the accent color. Window-control glyphs use a 1px stroke at 11px — Windows chrome convention.

Drawing new icons

Do Author on the 24×24 grid with 1.75 stroke, round caps/joins, 2px internal margins; test at 14px before accepting.
Do Add the glyph to the owning component's closed union — the icon name is part of that component's API.
Don't Import an icon library, use filled shapes, or encode state in the artwork (a struck-through variant like no-export is a separate glyph, not a style).
