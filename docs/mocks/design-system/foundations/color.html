Color

A dark workbench palette in the VS Code Dark Modern idiom: chrome on the darkest layer, content one step lighter, overlays lighter still. One blue accent carries all interaction; green, amber, and red are reserved for status semantics. Each dark value re-expresses a role the light implementation established — the mapping is recorded in tokens.css.

Surface stack

Derived from: editor-toolbar (#f9fafb), title-bar (#f3f4f6), panels (#ffffff), Monaco 'vs' (#fffffe), dialogs (#ffffff).

Elevation on the dark theme reads through surface lightening: the further a layer sits from the workbench frame, the lighter it is.

--tds-surface-chrome #181818 Title bar, activity rail, side panels
--tds-surface-editor #1f1f1f Editor and preview content plane
--tds-surface-raised #202020 Menus, pop-outs, tooltips
--tds-surface-overlay #242424 Modal dialog panels
--tds-surface-field #2a2a2a Inputs and selects on overlays
--tds-border #2b2b2b Structural hairlines (was #e5e7eb)
--tds-border-strong #3c3c3c Control borders (was #d1d5db)

Interaction — Trellis blue

Derived from: #3b82f6 (rail active, focus rings, dividers), #2563eb/#1d4ed8 (primary buttons), #93c5fd (soft outlines), rgba(59,130,246,…) tints.

The single accent family. It marks what is active (rail item, open document), what is focused (2px ring, −2px offset), what is primary (confirm buttons), and where a drag will drop. If something is blue, it responds.

Status

Derived from: connection-status (#22c55e / #ef4444 / #eab308 / #9ca3af), error-banner (#fef2f2 / #fecaca / #991b1b), danger actions (#b91c1c).

Status hues are reserved for connection state and operation outcomes. They are never used decoratively — a green anywhere in Trellis means "connected/succeeded", red means "failed", amber means "in between".

--tds-status-ok #3fb950 Connected dot
--tds-status-error #f85149 Disconnected dot, invalid borders
--tds-status-error-ink #ff7b72 Error text, danger row actions
--tds-status-error-surface #3a1d1d Error banner fill (border #6e2b28)

--tds-status-unknown #8b949e Idle / unknown dot
--tds-window-close #c42b1c Window-close hover — Windows convention, kept verbatim; not a palette member

Usage

Do Consume colors exclusively through tokens; the light→dark migration was possible because roles, not values, are referenced.
Do Keep rendered PlantUML diagrams on their white plate (.tds-preview__svg) — exports must stay legible outside the app.
Don't Use status green/amber/red for anything but connection and outcome semantics — no decorative status color.
Don't Introduce a second accent. If it isn't interactive, it isn't blue.
