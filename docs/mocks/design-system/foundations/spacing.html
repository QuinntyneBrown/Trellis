Spacing & layout

A 4px-base spacing scale for everything inside panels and dialogs, plus a small set of fixed chrome metrics that define the workbench frame itself. Density is a feature: rows are 4.8px-padded, panels are narrow, nothing floats in whitespace.

The workbench frame

Derived from: editor-page.component.scss, title-bar, editor-toolbar, resize-divider.

The frame is fixed-metric chrome; only the editor/preview ratio and the side panel width are user-adjustable, both within hard bounds.

TokenValueWhat it fixes
--tds-metric-titlebar-h35pxTitle bar height; the body is calc(100vh − 35px)
--tds-metric-rail-w48pxActivity rail width
--tds-metric-rail-button40pxRail button hit target (40×40)
--tds-metric-rail-icon20pxRail glyph box
--tds-metric-panel-min170pxSide panel minimum width
--tds-metric-panel-default260pxSide panel default width
--tds-metric-panel-max500pxSide panel maximum width
--tds-metric-divider1pxVisible resize seam
--tds-metric-divider-grab9pxInvisible grab area centered on the seam
--tds-metric-tree-action1.4remRow-action button (22.4×22.4)
--tds-metric-tree-action-icon14pxRow-action glyph box
--tds-metric-command-center-h24pxCommand-center pill height (max-width 38rem)
--tds-metric-window-control-w46pxEach window control button
--tds-metric-status-dot0.65remConnection dot diameter

The editor pane's share of the space right of the side panel is a ratio clamped to 0.2–0.8, default 0.5 (editor-page). The side panel clamps to 170–500px, default 260px.

Rules

Do Keep 40×40 (rail) and 1.4rem (row action) hit targets even when the glyph is smaller — the padding is the affordance.
Don't Introduce new fixed widths. New chrome adopts an existing metric or makes the case for a new token here first.
