Trellis Design System

The visual contract for Trellis — a dark, workbench-style language heavily inspired by the VS Code design system. Every token, component, and pattern here is derived from the current Angular implementation (frontend/src/app) and re-expressed on the Trellis dark theme. These pages are plain HTML/CSS: no build step, no dependencies, and they work over file://.

Principles

  • The workbench is chrome; the work is content. Chrome surfaces (title bar, activity rail, side panels) sit on the darkest layer; the editor and preview sit one step lighter. Nothing in the chrome competes with the document.
  • One accent, used sparingly. Trellis blue marks interaction — the active rail item, the open document, focus rings, primary actions, drop targets. It is never decorative.
  • Status color is semantic. Green, amber, and red are reserved for connection state and operation outcomes.
  • Derived, not invented. Every component page cites the Angular source it documents; metrics and states are carried over verbatim, only colors move to dark tokens.
  • Keyboard and screen-reader parity. Every pointer affordance has a visible :focus-visible treatment and an ARIA contract, documented per component.

Foundations

Color Surface stack, ink ramp, Trellis blue, status semantics, contrast. Typography UI and mono stacks, the type scale, uppercase-label rules. Spacing & layout 4px-base scale and the fixed workbench metrics. Elevation Surface layers, hairlines, shadows, scrims, z-index contract. Iconography Hand-authored 24×24 outline glyphs; rail and row-action sets. Motion Durations, easings, and where movement is allowed.

Components

Buttons Rail buttons, tree row actions, dialog and panel buttons. Form controls Text inputs, selects, labels, validation and upload affordances. Dialogs The one modal pattern: save, new document, move, export. Trees The shared tree-row language of the Explorer and Documents trees. Panels Side-panel anatomy, headers, empty states, resize dividers. Menus Raised lists: app menus, pop-out pickers, the command center. Toolbars & chrome Title bar, window controls, and the activity rail. Indicators Connection dot, spinners, error banners, toasts, badges.

Patterns

Workbench shell The full frame: title bar, rail, side panel, editor, preview. Editor surface Monaco (vs-dark) conventions: gutter, selection, cursor, metrics. Diagram preview The render lifecycle: placeholder, rendering, rendered, error. Feedback & empty states Empty trees, reconnect flows, save failures, native dialogs.

Using the system

Three stylesheets, three responsibilities:

FileOwnsPrefix
assets/tokens.css The authoritative design tokens — color, typography, spacing, shape, elevation, motion, z-index. The Angular theme consumes these values. --tds-*
assets/components.css The reference implementation of every reusable element, annotated with the Angular source it derives from. .tds-*
assets/docs.css Documentation-site chrome only — not part of the product design system. .ds-*

The light-theme screen mocks of the current app live one level up in docs/mocks; this site documents the target system those screens converge on.
