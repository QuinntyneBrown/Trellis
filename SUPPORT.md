# Support

Trellis uses GitHub issues for support and project coordination.

## Asking for Help

Before opening an issue:

- Search existing issues and pull requests.
- Check the [README](README.md), [Development Guide](docs/development.md), and [Testing Guide](docs/testing.md).
- Confirm whether the problem reproduces on the latest `main` branch.

When opening a support issue, include:

- Operating system and shell.
- .NET SDK version from `dotnet --version`.
- Node.js and npm versions.
- Java version from `java -version`.
- Browser name and version for frontend problems.
- Steps to reproduce and relevant logs.

## Bugs and Feature Requests

Use the GitHub issue templates for bugs and feature requests. Keep each issue focused on a single problem or proposal.

## Security Issues

Do not request security support in public issues. Follow [SECURITY.md](SECURITY.md).
